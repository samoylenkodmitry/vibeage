## Safe Migration Plan

Do these manually with sudo/root access; do not run deploy scripts.

1. Backup current state:
   - copy `/etc/nginx/sites-available/vibeage.eu`;
   - copy `/etc/nginx/sites-available/mail.dmitrysamoylenko.in`;
   - copy `/opt/vibeage/manage.sh`;
   - copy `/opt/vibeage-frontend/update-frontend.sh`;
   - preserve `/opt/vibeage/.env` and `/opt/vibeage-frontend/.env`.
2. Archive or inspect the untracked VPS-only files before cleaning them.
3. Move `/opt/vibeage` and `/opt/vibeage-frontend` to `origin/main` only after local changes are understood.
4. Rebuild the game container from `/opt/vibeage` only after confirming `docker-compose.yml` and env values.
5. Rebuild the frontend in `/opt/vibeage-frontend` with `NEXT_PUBLIC_GAME_SERVER_URL=https://vibeage.eu`.
6. Verify `/opt/vibeage-frontend/out/index.html` exists.
7. Run `nginx -t` before any reload.
8. Reload Nginx only if the test passes and mail vhost is still enabled.
9. Remove WireGuard only after confirming no current route or service depends on `wg0`.
10. Delete remote `server` branch only after the VPS is verified on `main`.
